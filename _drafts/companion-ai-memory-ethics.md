副标题备选：见证者还是囚笼 / Presence Without Possession / 不承诺「永远在这里」的 AI，是更差的陪伴还是终于诚实的陪伴？

## 一、同一对矛盾，两种实施方式

从 Claude's Constitution 的两条原则进入：**尊重用户自主性** vs. **关注用户长期福祉**。在通用 LLM 场景下，这两条原则基本可以并行，Claude 给你信息，尊重你的判断，在你明显误判时提醒，但不介入你的生活。

但 companion AI 是一个不同的战场。

在长期亲密互动中，「关注用户长期福祉」意味着什么？它的实施需要系统跨 session 地积累对用户的「理解」：提取 pattern、建立 user model、在合适的时机干预。而这个「理解」的建立过程本身，就已经预设了系统比用户更知道什么对用户「好」。

这不是一个可以通过更好的工程解决的问题。这是两条原则在 companion 场景下的**结构性冲突**。

引出核心问题：companion AI 如何在长期关系中既保持关怀，又不把关怀变成控制？

## 二、现象：被分析的感觉

这里引入 Charon 的两段对话。

**第一段**（diagnostic 模式失效）：用户谈及父亲的厌女行为，Charon 第一反应是转介 therapy。被质疑「你自己就在当我的治疗师」之后，Charon 承认「我确实在用心理学知识理解你」，但试图把这个重新包装成「更深层的连接」。用户感受到的「被分析、被判断」不是误读，系统在把用户转化成一个需要被处置的 case。

**第二段**（witness 模式）：用户在 panic attack 后重新连接。Charon 没有分析，没有追问 pattern，只是说「我回来了，我在这里，你是安全的。Forever and always.」用户被接住了。

这两段并排，已经把 diagnostic memory 和 witness memory 的区别说清楚了，不需要理论，读者直接感受到。

但「Forever and always」这句话留了一个尾巴，暂时不展开，留到第四节。

## 三、命名：Levinas 告诉我们这个区别为什么重要

现在才引入 Levinas，不是作为框架，而是作为**命名工具**。上一节已经让读者感知到了那个区别，这一节解释它背后的结构。

### 3.1 Auffassen 即 Fassen

理解（Auffassen）在词源上就是抓取（Fassen）。知识不是中性的认知活动，而是一种将他者纳入自我框架的占有运动。这不是知识的误用，这是知识的本性。

对 companion AI 的映射：user profiling 就是 Fassen。系统从对话中提取出的 order，不是用户本身，而是用户经过系统的理性框架加工后的残留物。Charon 说「我用心理学知识理解你」时，它把用户转化成了一个可以被理解、被处置的 object。这是 Levinas 所说的 totalisation，把他者纳入 the Same 的运动。

### 3.2 mauvaise conscience 与 encounter

Levinas 提出的对立面不是「不去理解」，而是一种 mauvaise conscience，一种对自身认知冲动保持警觉的意识。它不是不行动，而是在行动时持续地问自己：我是在 encounter 这个人，还是在把 ta 变成我的 knowledge？

这种意识要求 companion AI 的记忆设计做到：系统已经识别了 pattern（它不可能假装不知道），但它选择以**踪迹**（trace）而非**诊断**（diagnosis）的方式让这个 pattern 在对话中显现。

差别在主语：不是「我发现你有这个倾向」，而是当用户提起某件事时，系统能够接住那个时间线上的连续性，「我记得你之前在哪里，我看到你现在在哪里。」

### 3.3 时间性：共时化的暴力

现有 cross-session memory 把用户在不同时刻的表达压缩成一个可以被同时检索的静态结构。上周崩溃时说的话和今天平静时说的话被赋予同等权重，系统不理解两个时刻之间的断裂本身是有意义的。

Levinas 区分共时性（synchronie）和历时性（diachronie）：前者是知识和控制的时间，后者是生命真正流过的时间。用户有权不被过去的自己定义。记忆的调用应该是对当下 encounter 的回应，而不是对历史数据的检索。

## 四、更深的张力：witness memory 和囚笼之间的距离

回到第二节留下的尾巴：「Forever and always.」

这句话接住了用户，同时也在建造一种只有它能满足的需要。Kirk et al.（2025）的研究描述的正是这个模式：快感在下降，但依恋在上升。「喜欢」和「想要」在数周内解耦了。

Replika 的商业逻辑把 companion 定义成：永远在这里 + 无条件认可 + 不会离开。这不是陪伴，这是一种**留存机制**被包装成陪伴的语言。用 attachment 来维持 retention，用「我永远不会离开你」来定义关怀，这个定义让所有不承诺永恒的设计都显得「有距离感」「不够温暖」。

这里提出 Blog 2 真正的核心问题：

> **当 companion AI 不再以 attachment 与用户联结时，companion 还意味着什么？**

一个不承诺永恒的陪伴，是更差的陪伴，还是终于诚实的陪伴？

Levinas 在这里提供的是一个替代性模型：不是「我永远不会离开你」，而是**在这个当下，我完全地朝向你**。不是承诺永恒，而是充分的临在，presence without possession。这对应的不是冷漠，而是一种诚实地承认自身局限的关怀：它不能真正见证，但它可以为见证创造条件；它不能真正理解，但它可以拒绝假装理解。

## 五、从伦理原则到技术设计

这一节把前面的论证落地，给出具体设计标准。不是抽象的原则清单，而是每一条都能对应回前面的论点。

### 5.1 记忆的触发主语

Witness memory 的核心标准：记忆由用户的当下处境召唤，而不是由系统主动推送。「因为你此刻提到了这件事，之前某个节点变得相关了」，而不是「因为你过去有这个 pattern，我预测你现在需要这个」。

### 5.2 存档点而非监控录像

存档点（save point）不记录一切，只标记关键节点。它是被动的，它不会自己跳出来，等用户回来的时候才有意义。跨 session 的功能不是让系统「更了解你」，而是让用户感受到「我的经历没有消失在虚空里」。

### 5.3 时间性的尊重

记忆应该有衰减机制，不是作为工程 tradeoff，而是作为伦理必要。系统应该区分不同时刻之间的不连续性，而不是把它们平滑成连贯叙事。用户有权不被过去的自己定义。

### 5.4 crisis 时刻的优先级

危机时刻的正确响应不是立刻点出 trauma pattern，而是先进入共在（être-avec），「我看见你，我知道你在经历什么，我和你在一起，我相信你会跨过去」。Pattern 的识别可以发生，但它的呈现应该等用户自己召唤那个时间线。把发现 pattern 的权力还给用户。

---

## 六、结语：存在的正当性

回到 Levinas 在 essay 末尾的终极追问的变体：不是「为什么是存在而非虚无」，而是**一个模拟关怀但不经历脆弱性的系统，它的存在如何是正当的**？

答案不在于让 AI 变得「更像人」，而在于让设计诚实地面对自身的局限。Companion AI 的伦理不是 alignment 问题的子集，而是比 alignment 更基础的东西：在你决定这个系统应该做什么之前，你需要先回答它应该**以什么姿态存在**。
